Next.js 15 : ce qui change vraiment

A. Tanger Code
Développeur full-stack

Next.js 15 est arrivé avec son lot de changements. Plutôt que de lister chaque entrée du changelog, voyons concrètement ce qui va changer dans votre quotidien de développement et pour la performance de vos sites.

Server Actions stables

Les Server Actions passent en stable. Elles permettent d'écrire des mutations côté serveur directement depuis vos composants, sans créer d'API REST dédiée. Le code devient plus court et plus lisible.

async function createProject(formData) {
  'use server';
  await db.projects.insert({
    name: formData.get('name'),
  });
}

Un cache plus prévisible

Le comportement de mise en cache, longtemps source de confusion, devient explicite. Vous décidez clairement ce qui est mis en cache et pour combien de temps.

La meilleure optimisation, c'est celle que l'on comprend et que l'on maîtrise.

Faut-il migrer maintenant ?

Pour un nouveau projet, oui sans hésiter. Pour un projet existant, planifiez la migration : les gains en lisibilité et en performance en valent la peine, mais testez soigneusement le comportement du cache.

  • Adoptez les Server Actions progressivement
  • Auditez vos appels de données mis en cache
  • Mesurez vos Core Web Vitals avant / après

En résumé, Next.js 15 récompense ceux qui veulent un code plus simple et des performances maîtrisées.
